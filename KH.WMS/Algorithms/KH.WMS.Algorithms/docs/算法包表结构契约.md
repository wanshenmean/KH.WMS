# 算法包表结构契约（KH.WMS.Algorithms）

> 适用：`KH.WMS.Algorithms`（自洽 NuGet，不引用 `KH.WMS.Entities`）。
> 作用：声明本包**依赖的库表与列**。宿主改库结构前请对照本表——违反会导致包运行时 SQL 崩溃。
> 配套：[策略参数手册.md](./策略参数手册.md)

## 黄金法则

🔴 **不可变列**：被包查询实际引用（`Where` / `OrderBy` / `Join` / `GroupBy` / `Count` / `Select` 投影）。**改名 / 删除 / 改类型 → 运行时 SQL 报错（列名无效 / 类型转换失败）。** 需要变更须同步改包内 DTO 并升包版本。

⚪ **仅声明列**：DTO 里声明了、但当前查询未显式引用。改它一般无即时影响——**但若该表被"整行自动映射"查询（见下），仍属绑定**。如不确定，按🔴处理。

✅ **新增列**：随便加，包看不见（SqlSugar 只取 DTO 字段）。

> 一句话：**凡是 DTO 里列出的列，都不要改名/删除/改类型**；新增列无所谓。

## ⚠ 整行自动映射的表（整表绑定）

下列查询**没有显式 `Select`**，SqlSugar 会 `SELECT` 该 DTO 的**全部字段**——即 DTO 声明的每一列都被生成 SQL 引用，**全部按 🔴 对待**：

| 表 | 自动映射查询（无 Select） |
|---|---|
| `md_location` | `GetEmptyLocationsAsync` / `GetLocationsByZoneAsync` / `GetLocationsNearAsync` / `GetPairLocationAsync` / `GetFrontLocationStatusAsync`（`LocationQueryService`） |
| `md_warehouse_zone` | `GetStorageZonesAsync` / `GetZonesByWarehouseAsync`（`WarehouseQueryService`） |
| `md_aisle` | `GetAislesByZoneAsync` / `GetLeastLoadedAisleAsync`（`WarehouseQueryService`） |
| `md_logical_zone` | `GetLogicalZonesAsync`（`WarehouseQueryService`，虽含 Join 但取 `MdLogicalZoneDTO` 整行） |

→ 对这 4 张表，**DTO 所有列都是 🔴**（即便下表标 ⚪，也按不可变处理）。

其余表（`inv_inventory_detail/header`、`md_port`、`md_transfer_point`、`md_logical_zone_mapping`、`cfg_*`）的查询都带显式 `Select` 或只取标量，⚪ 列可较安全地删除。

## 变更影响速查

| 你要做的变更 | 影响 | 处置 |
|---|---|---|
| 给表**新增**列 | 无 | 直接加，包不感知 |
| **改名/删除** 🔴 列 | 运行时崩 | 必须同步改 DTO + 升包版本 |
| **改名/删除** ⚪ 列（非整行绑定表） | 一般无影响 | 建议同步 DTO |
| **改名/删除** 任意列（⚠整行绑定表：md_location/md_warehouse_zone/md_aisle/md_logical_zone） | 运行时崩 | 同步 DTO |
| **改 🔴 列类型** | 可能转换异常 | 同步 DTO 类型 + 回归 |
| 删表 | 全部依赖该表的策略失效 | 不可删 |

> 把"列被改"从运行时提前到测试期：建议加契约测试（对开发库跑各查询 / 校验 DTO 字段在 `information_schema` 存在）。
