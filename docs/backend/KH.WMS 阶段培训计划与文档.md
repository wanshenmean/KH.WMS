## 1. 培训目标

本培训不是简单讲一遍项目目录，而是让成员具备三类能力：

1. 能独立定位代码：知道需求应该落在哪个模块、哪个目录、哪个配置或哪个前端页面。
2. 能按项目方式开发：遵守后端模块边界、Contract 协作、前端 API 封装、权限与动态路由约定。
3. 能完成闭环验证：会启动、联调、查日志、跑构建、做基本验收，并能用文档查缺补漏。

培训结束后，成员应能独立完成一个中等复杂度业务功能：后端实体/接口/服务、配置或 Contract、前端页面/API/权限、联调与基础验收。

## 2. 培训对象与能力分层

| 对象 | 起点要求 | 培训后应达到 |
| --- | --- | --- |
| 后端新人 | 了解 C# 和 Web API 基础 | 能按项目模板新增业务接口，理解模块边界和事务 |
| 前端新人 | 了解 Vue 基础 | 能新增业务页面，接入 API、权限、字典和动态路由 |
| 全栈成员 | 能读懂前后端代码 | 能完成完整功能闭环和基础排错 |
| 实施/联调人员 | 能操作系统和看接口 | 能使用 Swagger、TraceId、文档定位常见问题 |

培训推进原则：

- 先全貌，后细节。
- 先开发约定，后业务流程。
- 先单端能力，后前后端联调。
- 先可运行，后可维护。

### 9.5 后台服务与定时任务

培训文档：

- `KH.WMS后端开发指引.md`（BackgroundServices 章节）
- `KH.WMS/KH.WMS后端代码架构评审报告.md`（相关评审条目）

培训目标：

- 理解 `IHostedService`（如 `DailySnapshotBackgroundService`）的注册与生命周期。
- 理解 `KH.WMS.QuartzJob` 的使用场景：每日库存快照、清理、报表等。
- 能新增或调整一个后台任务，并避免阻塞主请求线程。

关键知识点：

- 后台任务与业务 Service 的边界：业务逻辑放 Service，定时调度放 Job/HostedService。
- 任务失败重试与幂等：定时任务必须可重入，避免重复数据。
- 任务可观测性：日志、TraceId、与 MiniProfiler 的关系。

阶段产出：

- 能按业务流程解释入库、出库、库存、任务协作。
- 能判断跨模块调用是否需要 Contract。
- 能判断某个需求应配置化、策略化还是写业务代码。
- 能新增或调整一个后台任务（含 Quartz 与 HostedService 两种方式）。
