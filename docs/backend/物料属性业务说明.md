# 物料属性业务说明

> 本文档已按当前代码更新，描述 `MdMaterial` 中批管理、序列号管理、有效期管理和保质期字段的实际含义，以及这些属性在当前业务流程中的接入程度。
>
> 代码依据：`MdMaterial`、`MaterialController`、`MaterialContract`、入库单行、组盘明细、库存明细、前端物料管理页与入库组盘校验器。

## 1. 当前字段总览

实体：`KH.WMS.Entities.BaseData.MdMaterial`

表名：`md_material`

| 字段 | 类型/默认值 | 前端字段 | 当前含义 |
|---|---|---|---|
| `MaterialCode` | `string(50)`，唯一，必填 | 物料编码 | 物料主数据唯一编码 |
| `MaterialName` | `string(200)`，必填 | 物料名称 | 物料名称 |
| `CategoryId` | `long?` | 物料分类 | 关联物料分类 |
| `BaseUnitId` | `long`，必填 | 基本单位 | 关联物料单位 |
| `Status` | `int`，默认 1 | 状态 | `1` 启用，`0` 禁用 |
| `IsBatchManaged` | `int`，默认 0 | 批管理 | 标识该物料是否按批次管理 |
| `IsSerialManaged` | `int`，默认 0 | 序列号管理 | 标识该物料是否按序列号管理 |
| `HasExpiry` | `int`，默认 0 | 有效期管理 | 标识该物料是否有有效期 |
| `ShelfLifeDays` | `int?` | 保质期(天) | 物料默认保质期天数 |
| `TurnoverClass` | `string(1)?` | 周转分类 | A/B/C 等周转分类 |
| `Remark` | `string(500)?` | 备注 | 备注 |
| `ExtData` | `TEXT` | 扩展字段 | 由 `CfgExtField` 驱动的 JSON 扩展数据 |

前端页面：`KH.WMS.Client/src/views/basedata/material.vue`

接口入口：`MaterialController : ExtDataCrudController<MdMaterial>`，路由为 `/api/material`。

## 2. 与跨模块契约的关系

跨模块查询物料使用 `IMaterialContract`：

```text
GetByCodeAsync(string materialCode)
GetByCodesAsync(List<string> materialCodes)
```

当前 `MaterialInfo` 只返回：

| 字段 | 说明 |
|---|---|
| `Id` | 物料 ID |
| `MaterialCode` | 物料编码 |
| `MaterialName` | 物料名称 |
| `BaseUnitId` | 基本单位 ID |

也就是说，当前入库、出库等模块通过契约补全物料信息时，拿不到 `IsBatchManaged`、`IsSerialManaged`、`HasExpiry`、`ShelfLifeDays`。这些属性虽然已存在于物料主数据和前端表单中，但尚未通过契约参与跨模块业务判断。

## 3. 批管理（IsBatchManaged）

字段：`MdMaterial.IsBatchManaged`

取值：

| 值 | 含义 |
|---|---|
| `0` | 该物料不要求按批次管理 |
| `1` | 该物料要求按批次管理 |

业务含义：

```text
不开批管理：
  库存可按物料/容器/库位等维度管理，批次号可以为空。

开批管理：
  入库、组盘、库存、出库链路应携带 BatchNo，用于区分不同批次库存。
```

当前代码接入情况：

| 环节 | 当前实现 |
|---|---|
| 物料主数据维护 | 已有字段和前端开关 |
| 入库单行 | 有 `BatchNo` 字段 |
| 组盘 DTO/明细 | 有 `BatchNo` 字段，组盘时优先使用 DTO 值，否则继承入库单行批次 |
| 库存明细 | 有 `BatchNo` 字段和索引 |
| 出库单/分配 | 出库行和分配明细携带 `BatchNo` |
| 是否按物料属性强制批次必填 | 当前未接入 `IsBatchManaged` |
| 当前批次必填开关 | 使用全局配置 `CONTAINER.BATCH_MANAGEMENT` 控制组盘批次必填 |

重点说明：

当前“组盘必须填写批次号”的判断不是按单个物料的 `IsBatchManaged` 决定，而是由全局配置 `CONTAINER.BATCH_MANAGEMENT` 决定。若该配置为 `true`，所有组盘数据都要求 `BatchNo`；若为 `false`，即使物料 `IsBatchManaged = 1`，当前校验器也不会因此强制批次号必填。

## 4. 序列号管理（IsSerialManaged）

字段：`MdMaterial.IsSerialManaged`

取值：

| 值 | 含义 |
|---|---|
| `0` | 不按单品序列号管理 |
| `1` | 应按单品序列号管理 |

业务含义：

```text
不开序列号：
  库存按数量管理，例如数量 +5 或 -5。

开序列号：
  理论上每一件物料都应有唯一 SN，入库、出库、库存移动都要追踪到单品。
```

当前代码接入情况：

| 环节 | 当前实现 |
|---|---|
| 物料主数据维护 | 已有字段和前端开关 |
| 库存明细 | 有 `SerialNo` 字段和索引 |
| 入库单行/组盘 DTO | 未看到序列号字段 |
| 组盘校验 | 未看到序列号必填或唯一性校验 |
| 出库分配 | 当前策略对象主要携带物料、批次、数量，未看到按物料序列号强制分配 |

重点说明：

`IsSerialManaged` 当前更像主数据预留开关。库存明细表已预留 `SerialNo`，但入库组盘、出库分配、任务确认等关键路径尚未按该字段形成完整单品追踪闭环。

## 5. 有效期管理（HasExpiry）与保质期天数（ShelfLifeDays）

字段：

```text
MdMaterial.HasExpiry
MdMaterial.ShelfLifeDays
```

取值：

| 字段 | 含义 |
|---|---|
| `HasExpiry = 0` | 该物料不要求有效期管理 |
| `HasExpiry = 1` | 该物料有有效期管理要求 |
| `ShelfLifeDays` | 物料默认保质期天数，可用于由生产日期推算失效日期 |

业务含义：

```text
HasExpiry = 1 且 ShelfLifeDays = 365：
  理论上入库时应记录生产日期，并可计算 ExpiryDate = ProductionDate + 365 天。
```

当前代码接入情况：

| 环节 | 当前实现 |
|---|---|
| 物料主数据维护 | 已有 `HasExpiry` 和 `ShelfLifeDays` 字段，前端可编辑 |
| 入库单行 | 有 `ManufactureDate` 和 `ExpiryDate` 字段 |
| 组盘 DTO/明细 | 有 `ProductionDate` 和 `ExpiryDate` 字段 |
| 库存明细 | 有 `ProductionDate`、`ExpiryDate` 字段和有效期索引 |
| 有效期必填校验 | 使用全局配置 `CONTAINER.EXPIRY_MANAGEMENT`，不是按 `HasExpiry` 判断 |
| 自动计算失效日期 | 当前未看到由 `ShelfLifeDays` 自动计算 `ExpiryDate` 的实现 |
| 临期拒收/预警/冻结 | 当前未看到基于 `HasExpiry` 或 `ShelfLifeDays` 的完整实现 |

组盘有效期校验规则：

```text
CONTAINER.EXPIRY_MANAGEMENT = true
  -> 每条组盘数据至少要有 ProductionDate 或 ExpiryDate

CONTAINER.EXPIRY_MANAGEMENT = false
  -> 不执行该必填校验
```

重点说明：

当前 `HasExpiry` 和 `ShelfLifeDays` 没有直接驱动入库组盘校验。即使某个物料 `HasExpiry = 1`，如果 `CONTAINER.EXPIRY_MANAGEMENT = false`，当前校验器也不会强制该物料填写生产日期或有效期。

## 6. 与入库、组盘、库存的数据流

当前字段传递链路：

```text
入库单创建/收货
  InboundOrderLine.BatchNo
  InboundOrderLine.ManufactureDate
  InboundOrderLine.ExpiryDate

组盘
  ContainerBindDto.BatchNo / ProductionDate / ExpiryDate
  优先取 DTO 值；DTO 未传时继承 InboundOrderLine 对应字段

组盘明细
  InboundContainerBindDetail.BatchNo
  InboundContainerBindDetail.ProductionDate
  InboundContainerBindDetail.ExpiryDate

生成库存
  InventoryGenerationRequest.BatchNo
  InventoryGenerationRequest.ProductionDate
  InventoryGenerationRequest.ExpiryDate

库存明细
  InvInventoryDetail.BatchNo
  InvInventoryDetail.ProductionDate
  InvInventoryDetail.ExpiryDate
```

序列号字段当前主要出现在库存明细：

```text
InvInventoryDetail.SerialNo
```

## 7. 与全局配置的边界

物料属性是“物料主数据上的业务标识”，全局配置是“运行时校验/流程开关”。当前代码中，两者尚未完全打通。

| 业务目标 | 当前真正生效的控制点 |
|---|---|
| 组盘批次号必填 | `CONTAINER.BATCH_MANAGEMENT` |
| 组盘生产日期/有效期必填 | `CONTAINER.EXPIRY_MANAGEMENT` |
| 同容器禁止混料 | `CONTAINER.ALLOW_MIXED_MATERIAL = false` |
| 同容器禁止混批 | `CONTAINER.ALLOW_MIXED_BATCH = false` |
| 某个物料启用批管理 | `MdMaterial.IsBatchManaged` 仅存储/展示，当前未驱动校验 |
| 某个物料启用序列号管理 | `MdMaterial.IsSerialManaged` 仅存储/部分库存字段预留，当前未形成完整校验 |
| 某个物料启用有效期 | `MdMaterial.HasExpiry` 仅存储/展示，当前未驱动校验 |
| 由保质期自动算到期日 | 当前未看到实现 |

## 8. 典型场景

### 场景一：当前代码可支持的批次追踪

```text
背景：
  入库单行或组盘请求携带 BatchNo。

流程：
  入库单行记录 BatchNo
  -> 组盘时可覆盖或继承 BatchNo
  -> 组盘明细保存 BatchNo
  -> 后续生成库存时继续携带 BatchNo
  -> 出库分配和任务行可继续携带 BatchNo

配置影响：
  CONTAINER.BATCH_MANAGEMENT = true 时，组盘请求 BatchNo 必填。
```

### 场景二：当前代码可支持的容器混放控制

```text
严格模式：
  CONTAINER.ALLOW_MIXED_MATERIAL = false
  CONTAINER.ALLOW_MIXED_BATCH = false

效果：
  同一个 ContainerCode 下不能出现多个 MaterialId；
  同一个 ContainerCode 下不能出现多个 BatchNo。
```

### 场景三：有效期字段传递但未自动计算

```text
背景：
  食品物料 HasExpiry = 1，ShelfLifeDays = 30。

当前实际行为：
  系统不会自动根据 ShelfLifeDays 计算 ExpiryDate；
  需要入库或组盘数据显式传入 ProductionDate / ExpiryDate；
  若 CONTAINER.EXPIRY_MANAGEMENT = true，组盘至少要填写其中一个。
```

### 场景四：序列号管理尚未形成闭环

```text
背景：
  高价值设备 IsSerialManaged = 1。

当前实际行为：
  物料主数据可以记录该标识；
  库存明细有 SerialNo 字段；
  但入库组盘 DTO 未提供 SerialNo，当前流程不会强制逐件扫码。
```

## 9. 后续实现建议

若要让物料属性真正驱动业务规则，建议按以下方向补齐：

| 目标 | 建议改造 |
|---|---|
| 按物料控制批次必填 | 扩展 `MaterialInfo` 返回 `IsBatchManaged`，组盘校验按物料逐行判断 |
| 按物料控制有效期必填 | 扩展 `MaterialInfo` 返回 `HasExpiry`、`ShelfLifeDays`，有效期校验按物料判断 |
| 自动计算到期日 | 当 `HasExpiry = 1` 且有 `ProductionDate`、`ShelfLifeDays` 时自动生成 `ExpiryDate` |
| 序列号闭环 | 在入库、组盘、库存生成、出库确认中增加 `SerialNo` 或序列号明细表 |
| 序列号唯一性 | 对 `SerialNo` 增加唯一性规则或按物料唯一规则 |
| 物料属性与全局配置组合 | 全局配置控制是否启用规则，物料属性控制哪些物料适用规则 |

推荐组合规则：

```text
全局配置 = 是否启用这类校验能力
物料属性 = 当前物料是否适用这类校验能力

例如：
  CONTAINER.BATCH_MANAGEMENT = true
  且 MdMaterial.IsBatchManaged = 1
  -> 当前物料组盘时 BatchNo 必填
```
