## 1. 阅读路径

1. 先读 `KH.WMS项目技术栈与目录指引.md` 的配置与敏感信息约定。
2. 再读 `KH.WMS.Core-API-参考文档.md` 的配置指南。
3. 最后结合本文整理各环境配置。

## 2. 配置文件

| 文件 | 用途 |
| --- | --- |
| `KH.WMS/KH.WMS.Server/appsettings.json` | 默认配置结构和可提交默认值 |
| `KH.WMS/KH.WMS.Server/appsettings.Development.json` | 本地开发覆盖配置 |
| 环境变量/部署配置 | 测试、生产敏感配置 |

不要提交：

- 生产数据库连接串。
- 真实账号密码。
- 正式 JWT 密钥。
- 正式 License 私钥。
- 客户生产环境地址和凭据。

## 3. 常见配置项

| 配置 | 说明 |
| --- | --- |
| `urls` | 后端监听地址，当前默认 `http://*:9291` |
| `DbConnection` | 数据库连接、数据库类型、SQL 日志 |
| `Jwt` | Token 签名、Issuer、Audience、过期时间 |
| `Swagger` | Swagger 标题、路由、JWT 支持 |
| `MiniProfiler` | 性能分析开关、路由、存储 |
| `Cors` | 跨域策略 |
| `RateLimit` | 限流规则 |
| `License` | 授权校验 |
| `Logging` | 日志级别、文件路径、保留策略 |

## 4. 开发环境

开发环境建议：

- 使用 `appsettings.Development.json` 覆盖本机配置。
- 开启 SQL 日志和 Swagger。
- 使用本地或测试数据库。
- 不使用生产 License。
- 日志保留时间可以较短。

启动：

```powershell
cd D:\Git\0.KH.WMS\KH.WMS
dotnet run --project KH.WMS.Server\KH.WMS.Server.csproj
```

## 5. 测试环境

测试环境建议：

- 使用独立测试数据库。
- 配置接近生产，但保留 Swagger 访问能力。
- 日志级别至少保留 Warning/Error，联调阶段可临时打开 Information。
- 通过部署配置或环境变量注入敏感信息。

## 6. 生产环境

生产环境建议：

- 关闭不必要的调试输出。
- MiniProfiler 默认不对普通用户开放。
- Swagger 是否开放由团队安全策略决定。
- 数据库、JWT、License 等敏感配置走环境变量或安全配置中心。
- 日志目录、上传目录需要有写权限。

## 7. Windows Service

项目支持作为 Windows Service 运行。

培训时需说明：

- 本地开发仍可普通 `dotnet run`。
- 生产部署为服务时注意工作目录、日志目录、上传目录。
- 服务启动失败优先查 Windows 事件日志和 `Logs` 目录。

## 8. 静态文件与上传目录

后端可能暴露上传目录供附件下载。

检查项：

- 上传目录是否存在。
- 应用进程是否有读写权限。
- 静态文件访问路径是否符合安全要求。
- 不要暴露敏感目录。

## 9. 部署检查清单

- 后端能启动。
- Swagger 或健康入口可访问。
- 数据库连接成功。
- 登录接口可用。
- 日志能写入。
- 上传目录可读写。
- 关键业务接口可用。
- 前端代理或生产 API 地址指向正确后端。

## 10. 配置变更流程

配置变更建议流程：

1. 明确变更项和影响范围。
2. 在开发环境验证。
3. 在测试环境使用接近生产的数据验证。
4. 记录变更前后的配置值。
5. 安排生产变更窗口。
6. 变更后验证登录、关键接口、日志、上传下载。
7. 保留回滚方案。

高风险配置：

- 数据库连接。
- JWT Secret。
- License。
- CORS。
- 上传目录。
- 日志目录。
- 限流阈值。

## 11. 回滚与应急

部署前应准备：

- 上一个可运行包。
- 上一版配置。
- 数据库备份或变更脚本回滚说明。
- 服务重启方式。
- 日志查看路径。

常见应急：

| 问题 | 应急动作 |
| --- | --- |
| 服务启动失败 | 回滚配置，查启动日志 |
| 登录全部失败 | 检查 JWT/数据库/用户表 |
| 接口大量 500 | 查最近配置和日志 |
| 上传失败 | 检查目录权限和磁盘 |
| 前端跨域失败 | 检查 CORS 和网关 |
