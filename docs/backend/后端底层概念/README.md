# KH.WMS 后端底层概念专题索引

这组文档是 `KH.WMS后端开发指引 V3.0.md` 的“按概念查”入口。V3 总文档仍然是完整开发指引，这里拆出业务开发者必须理解的底层机制：启动扫描、模块边界、自动注入、AOP、请求管道、事务、CRUD、ExtData、跨模块契约、配置底座、校验器、编码状态、鉴权缓存、策略算法，以及 Swagger、JSON、MiniProfiler、License、限流、后台服务、登录加密等运行时能力。

这些文档不是框架源码说明书，重点是回答三个问题：

- 这个概念是什么，拿来解决什么问题。
- 业务开发什么时候需要用它，什么时候不要碰它。
- 出问题时应该从哪些代码位置和配置点排查。

## 按场景查

| 场景 | 先看 | 你需要确认什么 |
| --- | --- | --- |
| 新增一个普通业务 CRUD | [02](./02-模块边界与分层职责.md), [03](./03-依赖注入自动注册与AOP代理.md), [07](./07-CRUD基类与扩展钩子.md) | Controller 是否继承基类，Service 是否标 `[RegisteredService]`，业务逻辑是否放在 Service 钩子里 |
| Swagger 看不到接口 | [16](./16-Swagger与OpenAPI接口文档底座.md), [01](./01-启动入口与程序集扫描.md) | 模块程序集是否编译进输出目录，程序集名是否包含 `.Modules.`，Controller 是否在可扫描程序集里，构造函数依赖是否能解析 |
| 服务注入失败 | [03](./03-依赖注入自动注册与AOP代理.md) | 是否加注册特性，接口是否匹配，`ServiceType` 是否指定正确，生命周期是否合适 |
| AOP 日志、缓存、校验不生效 | [03](./03-依赖注入自动注册与AOP代理.md), [04](./04-AOP拦截器与特性使用.md) | 是否通过接口注入，是否被 `WithoutInterceptor` 排除，方法上是否加了对应特性 |
| 事务没回滚或重复开事务 | [04](./04-AOP拦截器与特性使用.md), [06](./06-数据访问仓储与事务边界.md) | 事务是 MVC Filter 或 `IUnitOfWork`，不是 Autofac AOP；确认外层事务边界 |
| 接口返回格式、字段名、日期或 TraceId 不对 | [17](./17-JSON序列化与HTTP契约约定.md), [05](./05-请求管道统一响应异常TraceId日志.md) | 返回值是否是 `ApiResponse`，JSON 是否 camelCase，null 是否被忽略，异常是否被全局 Filter 接管 |
| 主从表保存、导入导出、状态启停 | [07](./07-CRUD基类与扩展钩子.md) | `Navigate` 是否配置，实体是否实现 `IEnableDisableEntity`，是否重写了正确钩子 |
| 需要动态字段 | [08](./08-ExtData动态字段.md), [10](./10-Config配置底座.md), [17](./17-JSON序列化与HTTP契约约定.md) | 实体是否有 `ExtData` 字段，Controller 是否继承 `ExtDataCrudController`，扩展字段配置是否存在，`extDataRaw` 是否合法 |
| 跨模块调用库存、任务、状态配置 | [09](./09-跨模块Contract契约.md) | 是否只依赖 Contract 接口，是否避免直接引用对方模块 Service |
| 业务规则需要配置开关 | [10](./10-Config配置底座.md), [11](./11-可插拔校验器.md), [15](./15-应用配置Options与运行时配置边界.md) | 配置项应该放启动配置、Config 配置库还是系统参数；校验器是否按 `ValidatorCodes` 接入 |
| 生成单号或控制单据状态 | [12](./12-编码规则与单据状态.md) | 编码规则是否在配置库启用，状态流转是否由配置校验 |
| 登录、用户上下文、缓存 | [13](./13-鉴权缓存用户上下文与运行期开关.md), [22](./22-登录加密与密码哈希.md) | 匿名接口是否正确，当前用户是否可用，登录密码是否走 RSA 解密和哈希校验 |
| License 返回 402 | [19](./19-License授权许可与运行时拦截.md) | `License:Enabled`、白名单、License 文件、机器码、有效期和默认授权初始化 |
| 限流没有生效或误触发 429 | [20](./20-限流RateLimit底座.md) | 是否真正挂了 `UseRateLimiting`，是否读取 appsettings，限流 key 是用户、API Key 还是 IP |
| 接口慢、SQL 慢、方法链不清楚 | [18](./18-MiniProfiler性能观测底座.md), [04](./04-AOP拦截器与特性使用.md) | MiniProfiler 是否启用，SQL timing 是否出现，Service 是否经过 AOP 代理 |
| 后台任务、定时快照、无用户上下文 | [21](./21-后台服务作业与无请求上下文.md) | HostedService 是否注册，是否创建 scope，是否支持取消，是否有幂等检查 |
| 调用上架、货位分配、库存分配策略 | [14](./14-策略算法底座.md) | 是否通过 `IPolicyEngine` 调用，是否避免硬编码具体策略类 |

## 暂不纳入本组专题的能力

- `KH.WMS.Core.Modularity` 中的 `ModuleLoader` / `ModuleBase` 属于预留模块生命周期能力，当前主线业务开发主要依赖项目引用、程序集扫描和 `[RegisteredService]`。
- `Algorithms/KH.WMS.Engines/DataMap` 下有 DataMap、InterfaceMiddleware 和外部接口映射源码，但当前 `Program.cs` 中 `UseInterfaceMiddleware` 处于注释状态，本轮只在配置边界中提示，不单独成篇。
- `BusinessProcessorFactory` 是通用处理器工厂能力，当前未作为普通业务开发主路径，后续如果外部接口引擎启用，再单独整理。
