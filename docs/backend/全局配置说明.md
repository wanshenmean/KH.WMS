# 全局配置说明

> 本文档已按当前代码更新，描述 `cfg_global_config` 的数据结构、接口、前端管理页，以及当前业务代码实际读取的配置项。
>
> 代码依据：`CfgGlobalConfig`、`CfgGlobalConfigController`、`CfgGlobalConfigService`、`ConfigResolverContract`、入库组盘校验器与 `KH.WMS.Client/src/views/config/global-config.vue`。

## 1. 当前实现范围

全局配置存储在 `cfg_global_config` 表中，由 ConfigModule 管理。当前实现支持：

| 能力 | 当前代码状态 |
|---|---|
| 配置分组查询 | 已实现：`GET /api/global-config/groups` |
| 指定分组查询 | 已实现：`GET /api/global-config/group/{groupCode}` |
| 仅查 GLOBAL 值 | 已实现：`GET /api/global-config/value/{group}/{key}` |
| 分层解析配置值 | 已实现：`GET /api/global-config/resolve/{group}/{key}` |
| 按作用域查询配置 | 已实现：`GET /api/global-config/scoped/{groupCode}/{scopeLevel}` |
| 批量修改配置值 | 已实现：`PUT /api/global-config/batch` |
| 按分组重置默认值 | 已实现：`POST /api/global-config/reset/{groupCode}` |
| 手动新增配置 | 明确禁止，接口返回失败 |
| 删除配置 | 明确禁止，单删/批删均返回失败 |
| 配置值类型强校验 | 暂未看到后端强校验，前端按 `ValueType` 渲染控件 |
| 内置种子数据 | 当前仓库未发现完整种子脚本；配置项需由初始化脚本或数据库预置 |

> 注意：旧文档列出的部分配置属于业务规划项。当前业务代码真正读取的配置项集中在入库组盘校验，详见“当前被代码读取的配置项”。

## 2. 表结构

实体：`KH.WMS.Entities.Config.CfgGlobalConfig`

表名：`cfg_global_config`

| 字段 | 类型/约束 | 说明 |
|---|---|---|
| `ConfigGroup` | `string(30)`，必填 | 配置分组，如 `CONTAINER`、`INBOUND`、`OUTBOUND` |
| `ConfigKey` | `string(50)`，必填 | 配置键，如 `ALLOW_MIXED_BATCH` |
| `ConfigName` | `string(100)`，必填 | 页面展示名称 |
| `ConfigValue` | `string(500)`，必填 | 当前值，统一按字符串存储 |
| `DefaultValue` | `string(500)`，可空 | 重置默认值时使用 |
| `ValueType` | `string(20)`，默认 `STRING` | 支持 `BOOLEAN`、`NUMBER`、`STRING`、`ENUM`、`JSON` |
| `Options` | `TEXT`，可空 | 枚举候选值，JSON 数组字符串 |
| `Description` | `string(500)`，可空 | 描述说明 |
| `SortNo` | `int`，默认 0 | 分组内排序 |
| `Status` | `byte`，默认 1 | `1` 启用，`0` 禁用 |
| `ScopeLevel` | `string(30)`，默认 `GLOBAL` | `GLOBAL`、`WAREHOUSE`、`ZONE`、`DOC_TYPE` |
| `ScopeId` | `long?` | 作用域 ID，`GLOBAL` 时为空 |
| `Priority` | `int`，默认 0 | 同作用域内数值越大优先级越高 |

索引：

|---|---|
| `uk_group_key_scope` | `ConfigGroup`、`ConfigKey`、`ScopeLevel`、`ScopeId` |
| `idx_group` | `ConfigGroup` |
| `idx_scope` | `ScopeLevel`、`ScopeId` |
| `idx_status` | `Status` |

## 3. 配置作用域与解析规则

配置解析由 `ConfigResolverContract` 提供，支持按业务上下文逐级覆盖。

解析顺序：

```text
DOC_TYPE -> ZONE -> WAREHOUSE -> GLOBAL
```

解析入参：

| 参数 | 说明 |
|---|---|
| `group` | 配置分组 |
| `key` | 配置键 |
| `warehouseId` | 仓库级覆盖 |
| `zoneId` | 库区级覆盖 |
| `docTypeCode` | 单据类型级覆盖，代码会先通过 `CfgDocumentType.TypeCode` 查单据类型 ID |

解析规则：

1. 只读取 `Status = 1` 的配置。
2. `ScopeId` 必须与对应作用域匹配；`GLOBAL` 级要求 `ScopeId = null`。
3. 同一个作用域中按 `Priority` 降序取第一条。
4. 解析结果缓存 1 小时，缓存键形如 `config:{group}:{key}:w{warehouseId}:z{zoneId}:d{docTypeCode}`。
5. `GetConfigValueAsync(group, key)` 只查 `GLOBAL` 级，不执行分层覆盖。

缓存注意事项：

| 操作 | 当前行为 |
|---|---|
| `WarmUpAsync` | 预热所有启用的 `GLOBAL` 配置 |
| 批量修改/重置 | 只清理对应配置的全局缓存键 `w::z::d:` |
| 作用域配置修改后的缓存清理 | 当前未看到精确清理所有作用域缓存键的实现，涉及作用域配置时需注意缓存有效期 |

## 4. 当前被业务代码读取的配置项

当前仓库中通过 `ResolveConfigBoolAsync` 读取的配置项只有以下 4 个，均在入库组盘 `ContainerBindAsync` 的配置校验链中使用。

| ConfigGroup | ConfigKey | ValueType | 建议默认值 | 读取位置 | 业务含义 |
|---|---|---|---|---|---|
| `CONTAINER` | `BATCH_MANAGEMENT` | `BOOLEAN` | `false` | `BatchNoRequiredValidator` | 为 `true` 时，组盘数据必须填写批次号 |
| `CONTAINER` | `EXPIRY_MANAGEMENT` | `BOOLEAN` | `false` | `ExpiryDateRequiredValidator` | 为 `true` 时，组盘数据必须填写生产日期或有效期 |
| `CONTAINER` | `ALLOW_MIXED_MATERIAL` | `BOOLEAN` | `false` | `MixedMaterialValidator` | 为 `false` 时，同一容器不能绑定不同物料 |
| `CONTAINER` | `ALLOW_MIXED_BATCH` | `BOOLEAN` | `false` | `MixedBatchValidator` | 为 `false` 时，同一容器不能绑定不同批次 |

> 旧文档中曾把 `BATCH_MANAGEMENT` 放在 `INVENTORY` 分组，并使用 `SHELF_LIFE_MANAGEMENT`。当前校验代码实际读取的是 `CONTAINER.BATCH_MANAGEMENT` 和 `CONTAINER.EXPIRY_MANAGEMENT`，初始化数据应与代码保持一致。

## 5. 入库组盘校验关系

入口：`InboundContainerBindService.ContainerBindAsync`

声明的校验器顺序：

```text
BIND_DATA_NOT_EMPTY
BIND_QUANTITY
BATCH_NO_REQUIRED
EXPIRY_DATE_REQUIRED
MIXED_MATERIAL
MIXED_BATCH
```

配置相关校验：

```text
CONTAINER.BATCH_MANAGEMENT = true
  -> 每条 ContainerBindDto.BatchNo 必填

CONTAINER.EXPIRY_MANAGEMENT = true
  -> 每条 ContainerBindDto 至少填写 ProductionDate 或 ExpiryDate

CONTAINER.ALLOW_MIXED_MATERIAL = false
  -> 同一 ContainerCode 下的绑定明细不能来自不同 MaterialId

CONTAINER.ALLOW_MIXED_BATCH = false
  -> 同一 ContainerCode 下的绑定明细不能出现多个批次号
     批次号优先取 ContainerBindDto.BatchNo，未传时取 InboundOrderLine.BatchNo
```

校验失败会由 `ConfigValidationInterceptor` 在业务方法执行前短路，返回 `ServiceResult.Fail(...)`。

## 6. 管理页面行为

页面：`KH.WMS.Client/src/views/config/global-config.vue`

| 功能 | 当前行为 |
|---|---|
| 数据加载 | 调用 `/api/global-config/groups`，前端扁平化所有分组配置 |
| 搜索过滤 | 前端按 `configGroup`、`configKey`、`configName` 过滤 |
| 配置值编辑 | `BOOLEAN` 用开关，`ENUM` 用下拉，其余用输入框 |
| 批量保存 | 调用 `PUT /api/global-config/batch`，只提交发生变化的项 |
| 重置默认值 | 需先选择分组，调用 `POST /api/global-config/reset/{groupCode}` |
| 新增/删除 | 页面未提供入口；后端也禁止 |

前端展示依赖字典：

| 字典 | 用途 |
|---|---|
| `dict:config_group` | 配置分组展示 |
| `dict:config_value_type` | 值类型展示 |
| `dict:config_scope_level` | 作用域展示 |
| `dict:status_flag` | 状态展示 |

## 7. API 接口

专用接口：

| 方法 | 路由 | 说明 |
|---|---|---|
| `GET` | `/api/global-config/groups` | 按分组查询所有配置 |
| `GET` | `/api/global-config/group/{groupCode}` | 查询指定分组配置 |
| `GET` | `/api/global-config/value/{group}/{key}` | 查询 `GLOBAL` 级配置值 |
| `GET` | `/api/global-config/scoped/{groupCode}/{scopeLevel}?scopeId={id}` | 按作用域查询配置 |

## 8. 值类型说明

| ValueType | 存储形式 | 前端控件 | 备注 |
|---|---|---|---|
| `BOOLEAN` | 字符串，如 `true`/`false` 或 `1`/`0` | 开关 | `ResolveConfigBoolAsync` 仅把 `true` 或 `1` 解析为 `true` |
| `NUMBER` | 字符串数字 | 输入框 | 当前前端未使用专门数字控件 |
| `STRING` | 普通字符串 | 输入框 | 默认类型 |
| `ENUM` | 当前值为字符串，`Options` 为 JSON 数组 | 下拉框 | `OptionList` 由后端反序列化 `Options` 得到 |
| `JSON` | JSON 字符串 | 输入框 | 实体支持，但当前页面未提供 JSON 编辑器 |

## 9. 建议初始化配置

当前代码未发现完整种子数据，若需要让入库组盘校验生效，至少应初始化以下配置：

| ConfigGroup | ConfigKey | ConfigName | ConfigValue | DefaultValue | ValueType | Description |
|---|---|---|---|---|---|---|
| `CONTAINER` | `BATCH_MANAGEMENT` | 是否启用组盘批次校验 | `false` | `false` | `BOOLEAN` | 开启后组盘必须填写批次号 |
| `CONTAINER` | `EXPIRY_MANAGEMENT` | 是否启用组盘有效期校验 | `false` | `false` | `BOOLEAN` | 开启后组盘必须填写生产日期或有效期 |
| `CONTAINER` | `ALLOW_MIXED_MATERIAL` | 是否允许容器混料 | `false` | `false` | `BOOLEAN` | 关闭时同一容器只能绑定一种物料 |
| `CONTAINER` | `ALLOW_MIXED_BATCH` | 是否允许容器混批 | `false` | `false` | `BOOLEAN` | 关闭时同一容器只能绑定一个批次 |

可继续保留旧文档中的业务规划配置，但在未接入代码前应标记为“预留/未接入”，避免误以为修改配置后会影响运行逻辑。
