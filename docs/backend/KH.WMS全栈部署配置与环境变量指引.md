## 2. 环境矩阵

培训时先建立环境意识：同一套代码在开发、测试、生产中的差异主要来自配置、数据、网络和安全策略。

| 项 | 开发环境 | 测试环境 | 生产环境 |
| --- | --- | --- | --- |
| 前端入口 | `localhost:3000` | 测试域名或内网地址 | 正式域名或客户内网地址 |
| 后端入口 | `localhost:9291` | 测试 API 地址 | 生产 API 地址 |
| API 访问 | Vite proxy | 网关/反向代理 | 网关/反向代理 |
| 数据库 | 本地或开发库 | 独立测试库 | 生产库 |
| Swagger | 建议开启 | 可开启但限制访问 | 按安全策略决定 |
| MiniProfiler | 可开启 | 谨慎开启 | 默认关闭或严格限制 |
| 日志级别 | Debug/Information | Information | Information/Warning |
| 敏感配置 | 本地配置 | 环境变量/部署配置 | 安全配置/环境变量 |
| 数据清理 | 可重置 | 可构造测试数据 | 禁止随意操作 |

成员必须理解：不要用生产环境验证开发猜想，不要把本地配置提交成团队默认配置，不要把测试环境问题直接归因到代码。

## 3. 开发环境结构

开发环境通常是：

```text
浏览器
  -> http://localhost:3000
  -> Vite proxy /api
  -> http://localhost:9291
  -> KH.WMS.Server
  -> 数据库
```

端口：

| 服务 | 默认端口 | 配置位置 |
| --- | --- | --- |
| 前端 Vite | `3000` | `KH.WMS.Client/vite.config.js` |
| 后端 API | `9291` | `KH.WMS.Server/appsettings.json` |
| Swagger | `9291/swagger` | 后端 Swagger 配置 |
| MiniProfiler | `9291/profiler` | 后端 MiniProfiler 配置 |

## 4. 前端开发配置

前端开发代理：

| 前端路径 | 目标 |
| --- | --- |
| `/api` | `http://localhost:9291` |
| `/ws` | `ws://localhost:9291` |

开发约定：

- 前端代码只写 `/api`、`/ws`。
- 不在页面或 API 文件中写死后端完整地址。
- 换后端地址时只改 `vite.config.js` 或部署网关配置。

开发代理排查：

| 现象 | 检查 |
| --- | --- |
| `/api` 404 | Vite proxy、后端路由、Controller Route |
| 401 | Token、登录状态、JWT 配置 |
| CORS 报错 | 是否绕过代理直接请求后端 |
| `/ws` 连接失败 | WebSocket 代理、后端 WS 地址、网关支持 |

## 5. 后端开发配置

后端配置文件：

- `KH.WMS/KH.WMS.Server/appsettings.json`
- `KH.WMS/KH.WMS.Server/appsettings.Development.json`

常见配置：

- `urls`
- `DbConnection`
- `Jwt`
- `Swagger`
- `MiniProfiler`
- `Cors`
- `RateLimit`
- `Logging`
- `License`

后端配置变更原则：

- 能通过环境变量覆盖的，不把环境差异写死进代码。
- 生产密钥不放 `appsettings.json`。
- 开发默认值可以降低启动门槛，但必须与生产安全边界区分。
- 端口、数据库、JWT、CORS、限流、License 属于部署前必查项。

## 6. 测试环境建议

测试环境建议部署形态：

```text
测试域名
  -> 静态前端资源
  -> /api 反向代理到后端
  -> 后端服务
  -> 测试数据库
```

建议：

- 使用独立测试数据库。
- Swagger 可保留，但要控制访问范围。
- 前端构建产物使用测试 API 地址或反向代理。
- 后端敏感配置通过部署环境注入。
- 保留足够日志用于联调。

测试环境还要保留“可复现性”：

| 要求 | 说明 |
| --- | --- |
| 独立测试数据库 | 避免污染生产 |
| 测试账号固定 | 便于 E2E 和联调复现 |
| 日志可追踪 | 能按 TraceId 查问题 |
| 配置可回溯 | 知道当前环境使用哪个配置版本 |
| 数据可重置 | 关键流程可重复验证 |

## 7. 生产环境建议

生产环境建议：

- 前端使用 `npm run build` 产物部署。
- 后端以服务方式运行。
- 静态资源、API、WebSocket 路由由网关或反向代理统一转发。
- 生产数据库连接串不进入仓库。
- JWT 密钥、License、数据库密码走安全配置。
- MiniProfiler 和 Swagger 是否开放需按安全策略决定。

生产环境重点是稳定性和安全：

- 服务重启方式要明确。
- 日志目录、上传目录、临时目录权限要提前验证。
- 前端静态资源回退规则要提前配置。
- 数据库迁移或脚本必须可审计。
- 上线前确认是否需要停机窗口。

## 8. 敏感信息管理

不要提交：

- 生产数据库连接串。
- JWT Secret。
- License 私钥。
- 真实账号密码。
- 客户环境地址和凭据。
- 第三方接口密钥。

推荐来源：

- 环境变量。
- 部署平台配置。
- 安全配置中心。
- 仅本地存在的 Development 配置。

敏感配置检查表：

| 配置 | 风险 |
| --- | --- |
| 数据库连接串 | 泄露后可直接访问数据 |
| JWT Secret | 泄露后可能伪造 Token |
| License | 泄露或错误会导致授权失败 |
| 第三方密钥 | 泄露后可能产生费用或数据风险 |
| 客户地址和账号 | 泄露客户环境信息 |

## 9. 前端生产部署注意

前端构建：

```powershell
cd D:\Git\0.KH.WMS\KH.WMS.Client
npm run build
```

检查：

- 构建成功。
- 静态资源路径正确。
- 刷新动态路由页面不 404，服务端需回退到 `index.html`。
- `/api` 能转发到后端。
- `/ws` 能转发 WebSocket。

前端部署常见服务器规则：

| 路径 | 处理 |
| --- | --- |
| `/assets/*` | 返回静态资源 |
| `/api/*` | 反向代理到后端 |
| `/ws/*` | WebSocket 代理到后端 |
| 其他前端路由 | 回退 `index.html` |

## 10. 后端生产部署注意

后端构建：

```powershell
cd D:\Git\0.KH.WMS\KH.WMS
dotnet build KH.WMS.sln
```

检查：

- 服务能启动。
- 日志目录可写。
- 上传目录可写。
- 数据库连接可用。
- 登录接口可用。
- Swagger/健康入口按策略开放。
- Windows Service 工作目录正确。

Windows Service 场景重点检查：

- 服务账号是否有目录权限。
- 工作目录是否指向发布目录。
- `appsettings` 是否在发布目录中。
- 日志路径是否存在并可写。
- 端口是否被占用。
- 服务启动失败时能看到事件日志或 Serilog 日志。

## 11. 上线流程建议

上线建议按下面顺序执行：

1. 确认本次上线范围：前端、后端、数据库、配置、脚本。
2. 确认版本产物：前端构建包、后端发布包、配置文件、数据库脚本。
3. 在测试环境完成主流程验证。
4. 备份生产关键配置和数据库。
5. 停止或切流后端服务，按策略部署。
6. 部署前端静态资源。
7. 更新网关或反向代理配置。
8. 启动后端服务并检查日志。
9. 执行联合检查清单。
10. 保留观察窗口，确认无持续错误。

上线记录至少包含：

| 字段 | 说明 |
| --- | --- |
| 版本号/提交 | 可追溯代码 |
| 部署人 | 谁执行 |
| 部署时间 | 何时执行 |
| 配置变更 | 改了哪些配置 |
| 数据库变更 | 执行了哪些脚本 |
| 验证结果 | 哪些检查通过 |
| 回滚方案 | 出问题如何恢复 |

## 12. 回滚与应急处理

回滚前先判断问题类型：

| 问题 | 处理方向 |
| --- | --- |
| 前端静态资源错误 | 回滚前端构建包 |
| 后端启动失败 | 回滚后端发布包和配置 |
| 数据库脚本问题 | 按已评审回滚脚本处理 |
| 配置错误 | 恢复上一版配置 |
| 权限/字典配置错误 | 后台配置修正或回滚配置数据 |

应急原则：

- 先止血，再定位。
- 不在生产环境临时改代码。
- 每个临时配置变更都要记录。
- 回滚后仍要保留日志和错误样本。
- 生产数据修复必须有备份和复核。

## 13. 联合检查清单

- 前端首页可访问。
- 登录成功。
- Token 请求头正常。
- 菜单和动态路由正常。
- 列表接口正常。
- WebSocket 可连接。
- 上传/下载可用。
- 后端日志能按 TraceId 查到请求。
- E2E 关键流程通过或手工验证通过。

建议按业务再补充：

- 基础资料列表、新增、编辑。
- 入库主流程。
- 出库主流程。
- 库存查询。
- 任务分配和确认。
- PDA 关键动作。
- 报表或大屏页面。

## 14. 常见错误

| 错误 | 现象 | 正确做法 |
| --- | --- | --- |
| 前端生产刷新 404 | 动态路由无法刷新 | Web 服务器回退 `index.html` |
| API 地址写死 | 测试/生产无法切换 | 使用代理或环境配置 |
| WebSocket 未代理 | 实时功能失败 | 配置 `/ws` 转发 |
| 后端日志目录无权限 | 服务启动或写日志失败 | 配置目录权限 |
| 生产配置提交仓库 | 安全风险 | 敏感信息外置 |
